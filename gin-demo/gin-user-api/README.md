# 代码整体介绍
该代码是一个基于 Go 语言 Gin 框架实现的**简易用户管理 HTTP API 服务**，核心功能是提供用户的「单用户查询、用户创建、多条件筛选查询」能力，并通过中间件统计每个请求的耗时、生成唯一请求 ID，整体遵循 RESTful API 设计规范。

## 一、核心功能
| 接口路径          | 请求方法 | 功能描述                                                                 |
|-------------------|----------|--------------------------------------------------------------------------|
| `/api/user/:id`   | GET      | 根据路径参数 `id` 查询单个用户信息；校验 ID 格式，无匹配用户时返回提示     |
| `/api/user`       | POST     | 接收 JSON 格式的用户信息（姓名、年龄），创建新用户并存储到内存中           |
| `/api/user`       | GET      | 通过查询参数 `name`/`age` 筛选用户；无匹配结果时返回「查询失败」提示       |

## 二、核心技术知识点
### 1. Gin 框架核心使用
- **路由与分组**：
    - 用 `gin.Default()` 初始化 Gin 引擎（内置日志、崩溃恢复中间件）；
    - 通过 `r.Group("/api/user")` 创建路由分组，统一管理用户相关接口，简化前缀重复编写。
- **请求处理**：
    - 路径参数获取：`c.Param("id")`（获取 `/api/user/:id` 中的 ID）；
    - 查询参数获取：`c.Query("name")`/`c.Query("age")`（获取 `?name=xxx&age=18` 这类参数）；
    - JSON 请求体绑定：`c.ShouldBindJSON(&Newuser)`（解析 JSON 数据到结构体，自动校验参数）；
    - 响应返回：`c.JSON()`（返回 JSON 格式响应，支持自定义 HTTP 状态码）。
- **上下文操作**：
    - `c.Set("request_id", request_id)`：在请求上下文存储自定义数据（随机请求 ID）；
    - `c.MustGet("request_id").(int)`：从上下文读取数据并强制类型转换；
    - `c.Next()`：中间件中执行后续路由处理逻辑。

### 2. 中间件开发
- 自定义 `StatCost()` 中间件：
    - 统计请求耗时：`time.Now()` 记录开始时间，`time.Since(start)` 计算耗时；
    - 生成随机请求 ID：`rand.Intn(10000)` 生成唯一标识，关联每一次请求；
    - 全局注册：`r.Use(StatCost())` 让所有请求都经过该中间件。

### 3. 数据存储与处理
- 内存存储：用 `map[int]User` 模拟数据库，存储用户信息；通过 `last_user_id` 自增生成唯一用户 ID；
- 数据校验：
    - 结构体标签校验：`User` 结构体中 `binding:"required"`（姓名必填）、`binding:"gte=0"`（年龄≥0）；
    - 手动参数校验：校验 ID/年龄的字符串转数字（`strconv.Atoi()`），处理格式错误。

### 4. 错误处理与状态码规范
- HTTP 状态码使用：
    - `http.StatusOK (200)`：请求成功（查询/筛选成功）；
    - `http.StatusCreated (201)`：资源创建成功（用户创建）；
    - `http.StatusBadRequest (400)`：客户端参数错误（格式错/无筛选结果）；
    - `http.StatusNotFound (404)`：资源不存在（用户 ID 不存在）；
- 自定义错误提示：区分「参数格式错误」「资源不存在」「无筛选结果」等场景，返回友好的 JSON 提示。

### 5. Go 语言基础特性
- 结构体定义与标签：`User` 结构体通过 `json:"xxx"` 标签映射 JSON 字段，`binding` 标签做参数校验；
- 类型转换：`strconv.Atoi()` 实现字符串与整数的转换，处理参数格式；
- 切片与映射操作：用切片 `[]User` 存储筛选结果，用 `map[int]User` 存储用户列表；
- 条件判断与循环：遍历用户列表实现多条件筛选，判断结果集长度返回不同响应。

## 三、执行流程总结
1. 启动服务：初始化 Gin 引擎，注册全局耗时统计中间件，定义用户路由分组；
2. 接收请求：
    - 中间件先执行：记录开始时间 → 生成请求 ID → 执行后续逻辑 → 计算耗时并打印日志；
    - 路由处理：根据请求方法/路径匹配对应接口，处理参数、校验数据、操作内存存储；
3. 返回响应：根据处理结果返回 JSON 格式响应（成功/失败提示 + 数据/错误信息 + 请求 ID）。

## 四、关键特点
1. 轻量级：基于内存存储，无需依赖数据库，启动即可测试；
2. 规范性：遵循 RESTful API 设计，使用标准 HTTP 状态码；
3. 健壮性：包含多层参数校验，避免非法数据导致程序崩溃；
4. 可观测性：中间件统计每个请求的耗时、请求 ID，便于调试和日志排查。
