# Go + GORM + MySQL 任务实现总结
## 一、整体实现内容
基于 **GoLand** 开发、**DataGrip** 管理数据库，使用 Go + GORM 框架操作手动创建的 MySQL 数据库，完成**用户表（user）+ 帖子表（post）** 的完整数据操作实战，覆盖 GORM 核心用法与数据库基础规范。

## 二、数据表结构（DataGrip 手动创建）
### 1. user 用户表
- 存储用户基础信息，包含主键 ID、用户名、密码、创建/更新时间
- 用户名唯一，主键自增，具备基础约束

### 2. post 帖子表
- 存储帖子信息，包含主键 ID、标题、内容、关联用户 ID、创建/更新时间
- 通过 `user_id` 与用户表建立外键关联，一个用户可发布多篇帖子

## 三、分阶段实现功能
### 第一天：环境搭建与数据库连接
1. 搭建 Go 项目结构，引入 GORM 及 MySQL 驱动依赖
2. 完成 MySQL 数据库配置与连接初始化，实现项目与数据库的稳定连通
3. 定义与手动建表**完全匹配**的 GORM 模型结构体，绑定表名、字段、关联关系
4. 验证数据库连接正常，模型与数据表成功映射

### 第二天：单表基础 CRUD 操作
封装数据访问层（DAO），实现用户和帖子的**增、删、改、查**核心功能：
1. **创建**：新增用户、新增帖子
2. **查询**：根据 ID 单条查询、列表查询
3. **更新**：修改用户名、修改帖子标题/内容
4. **删除**：根据 ID 删除用户、删除帖子
5. 每一步操作均通过 DataGrip 验证数据正确性

### 第三天：关联查询 + 事务控制（核心进阶）
1. **一对多关联查询**
    - 查询用户时，自动加载该用户发布的所有帖子
    - 查询帖子时，自动加载对应的作者（用户）信息
2. **事务安全操作**
    - 实现事务方式创建帖子，保证数据操作原子性
    - 支持正常提交数据，也支持异常触发回滚（手动造错验证）
3. **自动绑定用户 ID**
    - 创建帖子时，通过传入用户 ID 自动赋值，完成用户与帖子的关联绑定

## 四、核心技术点实现
1. **GORM 基础**：数据库连接、模型映射、CRUD 封装
2. **关联关系**：用户与帖子的一对多关联查询
3. **事务机制**：开启事务、提交事务、异常回滚，保证数据安全
4. **工具配合**：GoLand 编码开发 + DataGrip 建表、数据验证

## 五、最终效果
1. 项目可独立运行，无报错，数据库操作稳定
2. 实现用户与帖子的完整数据管理能力
3. 掌握 GORM 操作 MySQL 的标准写法
4. 理解事务的作用与回滚机制，具备基础数据库开发能力
